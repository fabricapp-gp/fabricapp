📦 Overview

FabricIntel is a professional supply chain management and predictive analytics platform tailored for the fashion industry. It bridges the gap between sales forecasts and fabric fulfillment, allowing managers to identify supply risks before they impact production.

🚀 How to Run (Step-by-Step)

01 Environment Configuration

Create a .env.local file in the root with the following keys:

NEXT_PUBLIC_API_URL=http://localhost:8000 NEXT_PUBLIC_SENDER_EMAIL=your-gmail@gmail.com NEXT_PUBLIC_SENDER_PASSWORD=your-app-password # Firebase Credentials (from console) NEXT_PUBLIC_FIREBASE_API_KEY=... NEXT_PUBLIC_FIREBASE_PROJECT_ID=fabricapp
02 Backend Setup (FastAPI)

Install Python dependencies and start the API server:

cd api-server pip install -r requirements.txt python main.py

The backend will serve metrics and logic at localhost:8000.

03 Frontend Setup (Next.js)

Open a new terminal, install Node packages and start the UI:

npm install npm run dev

Access the app at http://localhost:3000.

✨ Core Features

1. Forecast Engine

Upload a standard sales CSV. The engine performs high-speed fuzzy matching to link sales keywords (e.g., "Bianca", "Onyx") to their respective fabric families. It generates a 14-day rolling demand model.

2. Control Tower (Dashboard)

A high-fidelity dashboard that aggregates reorder suggestions. You can manually adjust Inventory, WIP, and Lead Times to see instant "What-If" risk recalculations. Use the Manual Email button to alert vendors instantly.

3. Style Studio

Define the "recipe" for any garment. You can add Main Fabric 1, Main Fabric 2, and Lining (all in meters). Includes permanent delete and archive controls saved directly to Firestore.

4. Admin & Audit

Manage users (Admins vs Viewers). View a persistent log of every major action: who added a user, who archived a style, or who ran a forecast.

🏗️ Architecture Details

The app is built using a modern decoupled stack designed for scalability:

  • Frontend: Next.js 14 + Tailwind CSS + Lucide Icons.
  • Backend: FastAPI (Python) for heavy data processing.
  • Persistence: Firebase Firestore (NoSQL) for all user-generated state.
  • Routing: Vercel-ready /api remapping for cloud deployments.

🔧 Ongoing Maintenance

Cleaning Data

If the forecast becomes "Stale", simply upload a new Sales CSV in the Forecast Engine and click "Run Forecast". The dashboard will update globally.

User Access

Only Admins can delete users or modify persistent configurations. Ensure 2-Step Verification is enabled on your Gmail "App Password" to keep the alerting system working.
